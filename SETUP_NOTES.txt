IMPORTANT PATH NOTE (Windows quirk)
-------------------------------------
- file://path           -> FAILS on Windows (SurrealDB v3 path parser bug)
- file:///d:/...        -> FAILS (strips drive colon)
- rocksdb:///d:/path    -> WORKS  (RocksDB, matches server)
- surrealkv:///d:/path  -> WORKS  (SurrealKV, alternative)

NOTE: SurrealDB v3 ships with BOTH RocksDB and SurrealKV. RocksDB is NOT removed.
Use rocksdb:// to match the server engine exactly.

================================================================================

MODEL
-----
- File:        models/lgbm.onnx        (LightGBM regressor, 429 KB)
- Converted:   models/lgbm.surml       (SurrealDB ML format)
- Name:        close_prob  version 1.0.0
- Predicts:    sales close probability (0.0 – 1.0)
- Features:    29 float32 inputs (pre-normalised)
- Conversion:  python convert_to_surml.py

LOAD MODEL INTO DB (run once per server start after restart)
--------------------------------------------------------------
  curl -X POST http://localhost:8000/ml/import \
    -H "surreal-ns: vitasales" -H "surreal-db: vitasales" \
    -u "root:root" \
    --data-binary @models/lgbm.surml

APPLY FUNCTION + EVENTS (run once, persisted in RocksDB)
---------------------------------------------------------
  curl -X POST http://localhost:8000/sql \
    -H "surreal-ns: vitasales" -H "surreal-db: vitasales" \
    -u "root:root" \
    --data-binary @setup_ml.surql

MANUAL SCORE CALL
-----------------
  RETURN fn::score_lead(lead:<id>);

HOW IT WORKS
------------
1. DEFINE FUNCTION fn::score_lead($lead_id)
   - Joins: lead + conversation + message + product + organisation + generated_quotation
   - Encodes categoricals (pipeline stage, channel, product, industry, completeness)
   - Applies StandardScaler/MinMaxScaler normalisations (from training reference)
   - Calls ml::close_prob<1.0.0>([29 floats]) → raw float score
   - Returns score rounded to 4 decimal places

2. DEFINE EVENT score_on_lead_change ON TABLE lead
   - Fires on CREATE and on UPDATE (only when state/completeness/channel/org changes)
   - Writes score to lead.ai_confidence
   - Infinite-loop safe: ai_confidence changes do NOT re-trigger

3. DEFINE EVENT score_on_conv_change ON TABLE conversation
   - Fires on CREATE and on UPDATE (only when sentiment/summary/last_message_at changes)
   - Re-scores the linked lead (lead.ai_confidence)

SCORING VERIFIED
----------------
  lead state        ai_confidence
  NEW_LEAD          ~0.012
  SALES_QUALIFIED   ~0.124
  QUOTATION         ~0.203
  QUOTATION + POSITIVE conversation + summary  ~0.328
