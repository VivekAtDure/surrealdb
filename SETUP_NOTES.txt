IMPORTANT PATH NOTE (Windows quirk)
-------------------------------------
- file://path           -> FAILS on Windows (SurrealDB v3 path parser bug)
- file:///d:/...        -> FAILS (strips drive colon)
- rocksdb:///d:/path    -> WORKS  (RocksDB, matches server)
- surrealkv:///d:/path  -> WORKS  (SurrealKV, alternative)

NOTE: SurrealDB v3 ships with BOTH RocksDB and SurrealKV. RocksDB is NOT removed.
Use rocksdb:// to match the server engine exactly.

MANUAL SCORE CALLS
------------------
  RETURN fn::score_lead(lead:<id>);
  RETURN fn::score_consumer(lead:<id>);
  RETURN fn::score_interaction(lead:<id>);
  RETURN fn::score_product_quotation(lead:<id>);
  RETURN fn::score_lead_qualification(lead:<id>);

HOW IT WORKS
------------
1. DEFINE FUNCTION fn::score_lead($lead_id)
   - Joins: lead + conversation + message + product + organisation + generated_quotation
   - Encodes categoricals, applies StandardScaler/MinMaxScaler normalisations
   - Calls ml::close_prob<1.0.0>([29 floats]) → score rounded to 4 d.p.
   - Writes to lead.ai_confidence

2. DEFINE FUNCTION fn::score_consumer($lead_id)
   - Consumer engagement features from lead + conversation
   - Calls ml::consumer_score<1.0.0>([...]) → lead.consumer_score

3. DEFINE FUNCTION fn::score_interaction($lead_id)
   - Interaction quality features (message count, response times, sentiment)
   - Calls ml::interaction_score<1.0.0>([...]) → lead.interaction_score

4. DEFINE FUNCTION fn::score_product_quotation($lead_id)
   - Product fit + quotation features
   - Calls ml::product_quotation<1.0.0>([...]) → lead.product_quotation_score

5. DEFINE FUNCTION fn::score_lead_qualification($lead_id)
   - Lead completeness + profile features
   - Calls ml::lead_qualification<1.0.0>([...]) → lead.lead_qualification_score

EVENTS (auto-score on data change)
------------------------------------
  score_on_lead_change              ON TABLE lead         (→ ai_confidence)
  score_on_conv_change              ON TABLE conversation (→ ai_confidence)
  score_consumer_on_lead_change     ON TABLE lead         (→ consumer_score)
  score_consumer_on_conv_change     ON TABLE conversation (→ consumer_score)
  score_interaction_on_lead_change  ON TABLE lead         (→ interaction_score)
  score_interaction_on_conv_change  ON TABLE conversation (→ interaction_score)
  score_product_quotation_on_lead_change           ON TABLE lead              (→ product_quotation_score)
  score_product_quotation_on_quotation_created     ON TABLE generated_quotation (→ product_quotation_score)
  score_lead_qualification_on_lead_change          ON TABLE lead              (→ lead_qualification_score)

  All events are infinite-loop safe (score field changes do NOT re-trigger).
